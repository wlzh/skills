# 🔥 Code Roaster - 代码烤肉机

> **版本**: v1.0.0
> "If you can't stand the heat, get out of the kitchen!" - Gordon Ramsay

## 简介

Code Roaster 是一个用 Gordon Ramsay 风格毒舌吐槽代码质量的 Claude Code Skill。它会扫描你的代码，用幽默、犀利但专业的方式指出问题，让代码审查变得有趣又有价值！

## 特点

- 🔥 **毒舌但准确** - Gordon Ramsay 风格的犀利吐槽
- 🎯 **多维度分析** - 代码异味、Bug、安全、性能、最佳实践
- 😂 **搞笑且实用** - 在笑声中学习改进代码
- 📊 **详细报告** - 生成完整的 Markdown 格式报告
- 🎚️ **可调强度** - 温和/标准/残暴三种模式

## 安装

这个 skill 已经安装在你的 Claude Code 环境中。

## 使用方法

### 基础用法

```bash
# 烤当前目录的所有代码
/code-roaster

# 烤指定目录
/code-roaster ./src

# 烤单个文件
/code-roaster app.py
```

### 模式选择

```bash
# 温和模式（家庭友好，适合分享）
/code-roaster --mild

# 标准模式（默认，Gordon Ramsay 附体）
/code-roaster

# 残暴模式（火力全开，慎用）
/code-roaster --brutal
```

## 检测内容

### 🤢 代码异味
- 神秘变量名（a, b, data, temp）
- 拼音命名
- 超长函数
- 重复代码
- 深层嵌套

### 🐛 潜在 Bug
- 空值检查缺失
- 错误处理缺失
- 数组越界风险
- 异步操作裸奔

### 🔒 安全隐患
- SQL 注入风险
- XSS 漏洞
- 硬编码密码
- 敏感信息泄露

### 🐌 性能问题
- 低效循环（O(n²) 及以上）
- 内存泄漏风险
- 不必要的重复计算

### 📚 最佳实践
- 魔法数字
- 无用注释
- 缩进混乱
- 过时的 TODO

## 报告示例

烤制完成后会生成 `CODE_ROAST_REPORT.md`，包含：

- 📊 质量评分（0-10 分）
- 🏆 本次烤制亮点（最烂文件、最长函数等）
- 🔥 详细的逐文件分析
- 💬 每个问题的犀利吐槽
- 🎓 改进建议
- 💡 Gordon 的最后忠告

## 吐槽示例

**温和模式**：
> "这个变量名可以更清晰一点哦~ 建议改成 `userData` 而不是 `a`"

**标准模式**：
> "变量叫 `a`？你是在写密码学论文还是在侮辱未来的维护者？"

**残暴模式**：
> "变量叫 `a`？我见过的最懒的命名！你的键盘是不是只有这一个键能用？"

## 适用场景

✅ **适合**：
- 个人项目代码审查
- 团队代码质量提升
- 学习编程最佳实践
- 在 Twitter 上分享（温和模式）

❌ **不适合**：
- 给老板看（除非他有幽默感）

- 正式的代码审查文档（用标准工具）

## 常见问题

**Q: 吐槽会不会太毒舌？**
A: 可以用 `--mild` 温和模式，家庭友好。

**Q: 能检测所有的 bug 吗？**
A: 不能，这是静态分析工具，主要检测明显的代码问题和最佳实践违规。

**Q: 为什么我的完美代码被烤了？**
A: Gordon 说："没有完美的代码，只有还没被烤的代码。"

**Q: 报告可以分享吗？**
A: 当然！温和模式生成的报告特别适合发 Twitter。
